What is paired programming?

Paired programming is the practice of developing software with a team, usually focused on one person driving and another navigating.

Why is pair programming important to DBC?

Pair programming is important because software is rarely created alone. The work environment is replicated with pair programming and is important to DBC because it's mission to make students competent for the work setting. Pair programming also benefits DBC students by creating a benchmark on their performance by allowing them to compare their competencies and see how they are doing.

Driver vs Navigator

Pair progamming is done by delegating the roles of Driver and Navigator.

Driver description

The driver is the do-er; the one that writes the code as information is receieved by the navigator.

Navigator description

The navigator gathers and organizes information and techniques for the driver to implement. The navigator also seeks other resources for alternative solutions.
