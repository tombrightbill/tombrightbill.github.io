Post Five: Scopin' out Varibles

What on earth is variable scope? For something that sounds somewhat intimidating, the concept is remarkably simple. Variable scope defines where in a program a variable is accessible

Ruby has four types of variable scope; global, class, instance, and local. Let's break down each one

Global scope is the highest order scope. Any global variable is defined by the $ prefix. Any variable defined this way is accessible by any and all parts of a program regardless of where it is defined. Best practices discourage over use of this type of scope because the variable can be modified by any part of your program. This can make tracking bugs difficult

Class scope applies to a class as a whole. In my previous post, consider the 'Honda Accord' class. What is we wanted to know the total number of Accords that had been initialized? Clearly, this variable's scope is larger than just the instance, because we want it to apply to ALL Honda Accords. To do this, we use the @@ symbol to define a class variable.

As demonstrated in the last post, instance variables are created using the @ symbol and exist for each instance of a particular class. Local variables, as you might have guessed are only accesible within the method that calls them. For example, a variable within a loop or something similar. Cheers!
