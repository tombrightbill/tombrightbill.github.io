Post Four: You Classy Huh?

This post will cover classes, which can be most easily explained using a real world example. Let's say you drive a Honda Accord, and you've named your car Max. Your car has a number of attributes, shared amongst all Honda Accords. It get 25mpg, has a full tank capacity of 10 gallons, and can seat 5. You car Max, is a specific instance of a Honda Accord.

In Ruby, or any Object Oriented Language, the 'Honda Accord' would be the 'class' and you car 'Max' would be an instance of the class 'Honda Accord'. 'Max' would have a number of characteristics shared amongst all in the 'Honda Accord' class, but may also have some attributes that apply only to that instance, such as 'owner' and 'color'. Let's go ahead and create the 'Honda Accord' class to see what I mean.

Class Honda_Accord attr_reader :owner, :color, :mpg, :tank def initialize(owner, color) @owner = owner @color = color @mpg = 25 @tank = 11 end def drive_highway(hours) @tank-= (hours*50)/25 end def change_owner(new_owner) @owner = new_owner end end Max = Honda_Accord.new('Tom', 'British Racing Green')

So what we have done here is create a class called 'Honda_Accord' and assign certain attributes to the class. It has an owner attribute, which is assigned when an instance is created (i.e. 'Max'). Sae with the color attribute. We've got an mpg attribute, and a tank attribute (number of gallons it can hold). The funny '@' simply means that each instance of this class will have this variable when it is initialized. Additionally, we've defined some methods to go with our class so if I called Max.drive_highway(3) it would subtract (50 miles/hour * 3 hours)/25 miles per gallons = 6 gallons from the full tank so we would have 5 gallons left. The funky attr_reader simply means that certain instance variables can be called on and viewed. For example I can call Max.owner and Max.tank which will return 'Tom' and '5' respectively (assuming we go through with our Max.drive_highway(3) call).
