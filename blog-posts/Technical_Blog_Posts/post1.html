Post One: Thoughts on Display

A really challenging part of learning all this new information is the need to understand and memorize properties quickly. In order to create even the most basic web site, you'll need an understanding of a number of css properties. Today, I'm going to talk about the difference between display:inline vs display:inline-block.

In order to understand these two ways of displaying block elements inline, it's important to understand the two over arching structures that define html elements. An element can be one of two types, and inline element or a block element. So what distinguishes the two?

This paragraph is a block element. A block element can accept specifications like 'padding' and 'margin' both vertically and horizontally. However, it's defining characteristic is the line break that preceeds and follows it by default. A block element will also take up the full width available.

An inline element, such as a span is displayed 'inline' meaning it cannot accept vertical margin or padding (horizontal is acceptable) and is not followed by a line break. It will only take up as much width as needed. Additionally, it allows other elements to sit on it's left and right by default.

So with the definition of 'inline' and 'block' settled, let's move on to display:inline-block. This will take an element and display it 'inline' meaning it will not be preceeded and followed by a line break. It also will only take up as much width as needed. However, it CAN accept vertical and horizontal margin/padding.

An interesting thing to note about inline elements displayed this way, is that you can specify a padding both horizontally and vertically.
